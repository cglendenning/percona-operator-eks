DIAGNOSTIC TOOL

Run this diagnostic CLI to automatically detect which scenario is occurring:

detect-scenario CLI 
# From the percona_operator directory:
npx tsx dr-dashboard/cli/detect-scenario.ts --namespace percona

# Or with npm script:
npm run detect-scenario -- --namespace percona

# Options:
#   --namespace, -n   Kubernetes namespace to inspect (required)
#   --cluster-name, -c   PXC cluster name (optional, auto-detected)
#   --json            Output results as JSON
#   --verbose, -v     Show detailed command output
What it does:
  • Checks pod status, cluster health, and node availability
  • Tests API server connectivity, DNS, and service endpoints
  • Inspects PVC storage, replication, certificates, and backups
  • Identifies matching disaster scenarios with confidence levels
  • Provides direct links to recovery procedures

Requirements: KUBECONFIG must be set, kubectl installed
