DIAGNOSTIC TOOL

Run this diagnostic script to automatically detect which scenario is occurring:

detect-scenario.sh 
# From the percona_operator directory:
./dr-dashboard/cli/detect-scenario.sh --namespace percona --secret db-secrets

# Required:
#   -n, --namespace     Kubernetes namespace to inspect
#   -s, --secret        K8s secret with MySQL root password (must have 'root' key)

# Options:
#   -c, --cluster-name  PXC cluster name (optional, auto-detected)
#   -v, --verbose       Show detailed command output
#       --json          Output results as JSON
What it does:
  • Checks pod status, cluster health, and node availability
  • Tests API server connectivity and service endpoints
  • Queries MySQL for Galera cluster quorum status (read-only)
  • Inspects PVC storage, replication status, and backups
  • Identifies matching disaster scenarios with confidence levels
  • Provides direct links to recovery procedures

Requirements: KUBECONFIG must be set, kubectl and jq installed
